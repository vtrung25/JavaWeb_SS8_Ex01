
#### Phần 1 – Phân tích logic

1. **Lọt khoảng trắng vào Database**  
   `@NotNull` chỉ kiểm tra không được là `null`, nhưng vẫn cho phép chuỗi rỗng `""` hoặc chuỗi chỉ toàn khoảng trắng `"   "`.  
   Khi người dùng nhập dấu Space, `detailedAddress = "   "` → vẫn pass validation → lưu vào DB → nhãn dán in ra địa chỉ trắng.

2. **Không chặn được request từ Postman gây lỗi 500**  
   Controller dùng `@RequestBody AddressDto addressDto` nhưng không có `@Valid`.  
   → Spring bỏ qua toàn bộ Bean Validation (`@NotNull`).  
   Nếu hacker gửi `detailedAddress = null`, code bên trong (giả sử có logic xử lý) sẽ bị NullPointerException → lỗi 500 Internal Server Error thay vì trả về lỗi validation rõ ràng.
